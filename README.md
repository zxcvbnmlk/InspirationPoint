# InspirationPoint


Это клиентское SPA-приложение на Angular (версия 20) с использованием новых возможностей для оптимизации и ускорения загрузки.
В проекте использован архитектурный стиль Feature-based - группировка по фичам (страницам),
что позволяет легко ориентироваться в структуре проекта и поддерживать его.
В проекте отделена бизнес логика в сервисе для чистой структуры и читаемости. 
Используется lazy loading для оптимизации загрузки страниц и standalone компоненты для упрощения структуры 
и уменьшения количества модулей. Для элементов формы UI используется Angular Material (MIT License - свободная лицензия), 
что обеспечивает единый дизайн и контроль отображения в разных браузерах (это промышленный вариант,
поскольку браузер не даёт полный контроль над стандартными элементами), можно сделать полностью кастомные элементы,
но это займёт больше времени и усложнит дальнейшую разработку.

Для Обработки ошибок глобально используется ErrorInterceptor, он перехватывает все запросы,
в дальнейшем можно использовать для логирования ошибок. Также есть перехват ошибок на уровне компонента,
который может отображать пользователю сообщение об ошибке (предложу добавить всплывающие сообщения)

technology stack: 
Angular 20,
node v20.19.3
npm 10.8.2

Если у вас нет Node.js, Angular CLI.
Установка Node.js
Скачать и установить:
👉 https://nodejs.org/

затем установить Angular CLI глобально:
```
npm install -g @angular/cli
```
После установки проверить:
```
node -v
npm -v
```

To start a local development server, run:

```
git clone https://github.com/zxcvbnmlk/InspirationPoint.git
cd InspirationPoint
npm install
ng serve
```

Дополнения:
- в запросе событий `/restapi/events`  не передается регион 
- в запросе событий `/restapi/events`  нет пагинации (постраничной передачи данных). Должны быть параметры номер страницы и количество элементов на странице.
- если делаем пагинацию на сервере, то нужно делать сортировку по дате события, чтобы при добавлении новых событий не нарушалась последовательность отображения. Сначала должны отображаться самые новые события.
- сервер присылает дату "2025-07-10 00:15:00+05:00" - это не стандарт ISO формат для JS. Нужно 2025-07-10T00:15:00+05:00. Из-за этого браузер не правильно парсит дату и нарушается сортировка.
